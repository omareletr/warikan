# Plan: Sticky Avatar Row on AssignPage

## Status
- [x] Phase 1: Make header + avatar row sticky (committed & pushed)
- [ ] Phase 2: Frosted glass styling for sticky header

## Problem
When a receipt has many line items, users must scroll the entire dish list on the AssignPage. The person avatar row scrolls away with the page content, forcing users to scroll back to the top to switch the selected person before scrolling back down to continue assigning dishes.

## Solution
Make the page header (back button + title) and the person avatar row sticky at the top of the viewport on the AssignPage. The "Tap to assign to X" instruction text and "Assign All" button remain in-flow and scroll normally.

## Scope
- **AssignPage only** вҖ” other pages don't have the same two-axis selection problem.

---

## Phase 1: Sticky positioning (DONE)
- Wrapped header + avatar row in `sticky top-0 z-10 bg-background` container
- Removed `ScrollArea` in favour of natural page scroll (required for `sticky`)
- Added `border-b border-border/20` separator
- Moved `px-6` to a new content wrapper for the dish list

## Phase 2: Frosted glass card for avatar row only

### Goal
Separate the sticky header into two visual zones:
- **Header** (back button + title): plain solid `bg-background` вҖ” no glass effect
- **Avatar row**: frosted glass card with `rounded-3xl` corners matching the bottom CTA bar

### Changes in `app/split/assign/page.tsx`

**1. Update the outer sticky container (line 100):**

Replace the current frosted glass styling with a plain background:
```
Before: bg-card/80 backdrop-blur-xl px-6 pt-14 pb-2 border-b border-border/30 shadow-md shadow-black/10
After:  bg-background px-6 pt-14 pb-4
```

- `bg-background` вҖ” solid dark background so dish items don't show through behind the title
- `pb-4` вҖ” enough padding below the avatar card before content starts
- Remove `border-b`, `shadow-md` вҖ” the avatar card's own border/shadow handles separation

**2. Wrap the avatar scroll in a frosted glass card (lines 106-113):**

Replace the current full-width avatar `<div>` with a card container + inner scroll:
```tsx
// Before:
<div
  className="-mx-6 mt-6 flex gap-5 overflow-x-auto px-7 py-2 pb-3"
  style={{ maskImage: "..." }}
>
  {avatars}
</div>

// After:
<div className="mt-4 -mx-2 rounded-3xl border border-border/30 bg-card/80 backdrop-blur-xl shadow-md shadow-black/10">
  <div
    className="flex gap-5 overflow-x-auto px-5 py-3"
    style={{ maskImage: "linear-gradient(to right, transparent, black 24px, black calc(100% - 24px), transparent)" }}
  >
    {avatars}
  </div>
</div>
```

Key properties on the outer card:
- `rounded-3xl` вҖ” matches bottom CTA bar's border radius (24px)
- `border border-border/30` вҖ” matches bottom bar's border
- `bg-card/80 backdrop-blur-xl` вҖ” frosted glass, same as bottom bar
- `shadow-md shadow-black/10` вҖ” softer than bottom bar since it's not floating
- `-mx-2` вҖ” slight negative margin so the card extends close to screen edges while keeping rounded corners visible

Key adjustments on the inner scroll:
- `px-5` вҖ” horizontal padding inside the card (smaller than the previous `px-7` since the card is already inset)
- `py-3` вҖ” vertical padding inside the card
- `maskImage` gradient shortened to `24px` to match the card's internal padding
